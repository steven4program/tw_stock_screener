# 台股選股器 — UI 設計簡報（給 Claude Design）

> 用法：把這份簡報整段貼給 Claude Design，並附上 `sample-snapshot.json`（真實示意資料）。**不要餵整個程式碼倉庫**——這份簡報已含設計所需的全部脈絡。請它輸出「元件＋狀態」為單位的視覺稿與一組 design tokens。

---

## 1. 這是什麼產品

一個**台股選股器網頁**：每天收盤後自動掃描上市＋上櫃股票，列出符合兩種技術條件（A 季線型／B 月線型）的股票，並說明每檔「為什麼入選」。使用者打開網頁就是看「今天選出哪些股票」。**唯讀、無登入、無下單**。

## 2. 使用者輪廓與無障礙約束（最重要）

- 主要使用者：**65 歲、非工程師**。可能在手機或平板上看。
- 設計準則：
  - **大字體**：內文 ≥ 18px、重要數字（股名、百分比、連買天數）更大更粗。
  - **高對比**：符合 **WCAG 2.1 AA**（一般文字對比 ≥ 4.5:1，大字 ≥ 3:1）。避免細灰字。
  - **大觸控目標**：按鈕/可點區 ≥ 44×44px；參數 +/− 鈕要大顆好按。
  - **狀態一目了然**：用顏色＋圖示＋文字三重提示（不要只靠顏色，色盲也要懂）。
  - **資訊密度低**：寧可少而清楚。手機上表格轉為**卡片**。
  - 繁體中文介面。漲用紅、跌用綠（台股慣例，與歐美相反）。

## 3. 畫面資訊架構（單一主頁，由上到下）

> 這是「畫什麼、放哪裡」。演算法細節不需要，請勿設計參數以外的進階控制。

1. **頂部更新狀態列**（最顯眼）
   - 成功：`✅ 今日已更新 ・ 資料日期 2026-06-09`
   - 尚未更新（沿用舊資料）：`⚠️ 今日尚未更新（沿用 2026-06-06 資料）`
   - 部分成功：加註 `⚠️ 董監資料沿用 2026-04 月份`
   - 失敗：`⛔ 今日更新失敗（顯示為上次成功資料 2026-06-06）`
2. **參數摘要 / 調整區**
   - 兩個可調參數，大顆 **+ / −** 即時調整：
     - `法人連買天數 N`（範圍 1–10，預設 2）
     - `董監持股門檻 X%`（範圍 5–50，預設 15）
   - 固定參數只顯示不可調：距均線 0~10%、月線 20MA、季線 60MA、扣抵 5 個交易日。
   - 一併顯示資料日期。
3. **結果統計**：`全部 N 檔 ・ A 季線型 N 檔 ・ B 月線型 N 檔 ・ A+B N 檔`。
4. **分頁（Tab）**：`全部` / `A 季線型` / `B 月線型` / `A+B 同時符合`。
5. **股票清單**（桌機表格 / 手機卡片）：每檔一列/一卡，可**展開**看「為什麼入選」。
6. **排序控制**：預設綜合排序；可手動切換：連買天數 / 距均線% / 買超張數 / 成交量 / 董監持股%。
7. **頁尾**：固定顯示免責聲明摘要（本工具僅為個人選股資訊整理，不構成投資建議；資料可能延遲/缺漏）。

## 4. 一檔股票要顯示的內容（對照 `sample-snapshot.json` 的真實欄位）

- **代號 + 名稱**（如 `9921 巨大`）— 最大最粗。
- **類型標籤**：`A 季線型` / `B 月線型` / `A+B`（A+B 要最醒目，例如實心徽章）。
- **收盤價** 與 **漲跌幅%**（紅漲綠跌）。
- **連買天數**（如「連買 5 天」）。
- **買超張數**（三大法人合計，如 1,850 張）。
- **董監持股%**，旁邊小字標示**資料月份**（如 `17.1%・2026-04`）。當某檔月份較舊（如 2026-03），要看得出「資料較舊」的提示。
- **成交量**（張）。
- **均線資訊**（依分頁顯示）：均線值、距均線%、均線狀態（「已上彎」或「扣抵向上」）。
  - `A 季線型` 分頁：顯示季線（60MA）那組。
  - `B 月線型` 分頁：顯示月線（20MA）那組。
  - `A+B` 分頁：**同時顯示月線與季線兩組**。
  - `全部` 分頁：A+B 列顯示兩線；A 列顯示季線、B 列顯示月線；不適用處以「—」。
- **展開後**：逐條「為什麼入選」（每條一句，見 sample 的 `reasonsA`/`reasonsB`，通常 4 條）。A+B 同時展示 A、B 兩組原因。

## 5. 必畫的所有狀態（最容易被忽略）

1. 正常成功（多檔結果）— 以 `sample-snapshot.json` 為內容。
2. **今日尚未更新**（頂部黃色提示，沿用舊資料日期）。
3. **部分成功**（頂部提示「董監資料沿用 2026-04 月份」）。
4. **更新失敗**（頂部紅色提示，仍顯示上次成功資料）。
5. **空清單**（某分頁/某參數下 0 檔）—「今日無符合條件的股票」友善空狀態。
6. 手機**卡片**版（同內容、轉為卡片堆疊）。
7. 參數調整中的即時回饋（調 N/X 後清單與統計瞬間變化）。

## 6. 內容真實感（請用附檔資料設計，勿用 lorem ipsum）

`sample-snapshot.json` 提供 5 筆真實示意列，含 A+B / A / B、長中文股名、四條入選原因長句、不同董監資料月份。請依這些**實際內容長度**排版（中文字較寬、原因句較長、數字要對齊）。

## 7. 請輸出

- **桌機主頁**（含頂部狀態列、參數區、統計、分頁、表格、展開原因、排序、頁尾）。
- **手機卡片版**主頁。
- **上述 5 種狀態**的視覺（成功/未更新/部分成功/失敗/空清單）。
- 以**元件**拆解：狀態列、參數控制、統計列、分頁、股票列/卡片（含展開態）、排序控制、頁尾免責。
- 一組 **design tokens**：色票（含紅漲綠跌、狀態色）、字級階（標題/數字/內文/小字）、間距、圓角、陰影 —— 皆滿足 WCAG AA 與大字體/大觸控目標。

> 拿到設計稿後，會由「計畫 3：前端」依元件一對一實作，資料來自 `/api/snapshots/latest` 再經 `runFilter` 篩選。
