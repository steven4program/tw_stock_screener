# 董監持股資料 POC — 結論

## 判定：**GO**（上市無條件通過；上櫃通過，附「更新頻率」一項待續觀察的緩解措施）

可從免費公開來源，每月取得並解析出每檔上市＋上櫃股票的「全體董監持股 %」，且數值已與公開資訊觀測站（MOPS）官方值**逐股完全相符**。採「批次明細檔 ÷ 已發行股數」一次抓全市場；MOPS 作為驗證 oracle 與備援。唯一殘留風險：上櫃明細檔在 data.gov 被標示為「每1年」，需於管線加「資料年月新鮮度守門」並保留 MOPS 每月備援（已驗證可行）。

## 選定來源（分上市/上櫃）

主來源為 **TWSE / TPEx OpenAPI 的「董監事持股餘額明細」批次檔**（一次取全市場，免逐檔爬）。以 `Accept: text/csv` 取得 CSV（預設回 JSON）。

| 項目 | 上市（TWSE） | 上櫃（TPEx） |
|---|---|---|
| 董監明細端點 | `https://openapi.twse.com.tw/v1/opendata/t187ap11_L` | `https://www.tpex.org.tw/openapi/v1/mopsfin_t187ap11_O` |
| 取 CSV 方式 | 請求標頭 `Accept: text/csv` | 請求標頭 `Accept: text/csv` |
| data.gov.tw 對應資料集 | dataset **22811** | dataset **22812** |
| 宣告更新頻率 | **每1月（monthly，已由 data.gov REST 確認）** | **每1年（annual）** ⚠️ 見風險 |
| 最新可得月份 | **2026-04**（資料年月 `11504`，出表日期 `1150520`） | **2026-04**（資料年月 `11504`，同上） |
| 明細欄位 schema | 中文欄名 | 中文欄名（與上市相同） |
| 編碼 | UTF-8（無 BOM、CRLF、欄位加引號） | UTF-8 |
| 明細不重複公司代號數 | 1,077 | 887 |

**發行股數來源（採「Σ董監股數 ÷ 已發行股數」法時必需，兩市場 schema 不同）：**

| 項目 | 上市（TWSE） | 上櫃（TPEx） |
|---|---|---|
| 公司基本資料端點 | `https://openapi.twse.com.tw/v1/opendata/t187ap03_L` | `https://www.tpex.org.tw/openapi/v1/mopsfin_t187ap03_O` |
| 公司代號欄位 | `公司代號`（中文） | `SecuritiesCompanyCode`（**英文**） |
| 已發行股數欄位 | `已發行普通股數或TDR原股發行股數` | `IssueShares`（**英文**） |

> **直接取「已發行股數」欄位，不用「實收資本額 / 10」估算**（非 10 元面額、TDR 原股、特別股、私募都會讓估算失真）。2330 實測已發行股數 = 25,932,370,067 股。
>
> 注意上櫃**基本資料**（`t187ap03_O`）為英文 schema，但上櫃**董監明細**（`t187ap11_O`）為中文 schema —— 兩者欄名規則不同，程式須分開處理。

**MOPS（驗證 oracle ／ 上櫃每月備援，架構 A）：**
- 端點：`POST https://mopsov.twse.com.tw/mops/web/ajax_stapap1`，body `co_id=<代號>&year=<民國年>&month=<MM>&step=1&firstin=1`。
- ⚠️ 舊主機 `mops.twse.com.tw` 已停用（回安全性封鎖頁）；改版後主機為 **`mopsov.twse.com.tw`**。
- 回應為單一公司的董監明細 HTML，內含官方彙總列 **`全體董監持股合計`（股數，非百分比）**，且為**去重、僅董監**的權威值。

## 彙總方法（關鍵——明細檔是「全體內部人」，非僅董監）

`t187ap11` 明細**列出全體內部人**（董事、監察人、獨立董事、**並含**總經理／副總經理／經理／協理／大股東／財務及會計主管，以及**法人代表人**列）。若天真地把每列 `目前持股` 相加會嚴重高估（6488 環球晶 → 140%，不可能）。

**正確規則（已驗證對齊 MOPS `全體董監持股合計`）：**
1. **只計董監本人列**：`職稱` 含「董事」或「監察人」**且不含「法人代表」**（→ 董事長本人／董事本人／獨立董事本人／監察人本人計入；董事之法人代表人、監察人之法人代表人**排除**；總經理／大股東／經理人／財會主管因不含「董事/監察人」而**排除**）。
2. **依姓名（實體）去重**：法人董事占多席時每席重複列出相同持股，該實體只計一次。

實作於 `src/sources/datagov.ts` 的 `parseDirectorRows()`，再以 `src/aggregate.ts` 的 `aggregateByShares(rows, 已發行股數)` 得百分比。

兩者皆 ≥ 95%。上市缺漏 13 檔幾乎全為 **TDR（910322／910861／911608／911622／911868／912000）與外國發行人（4582／7803／7827／9103／9105／9110／9136 等 KY/F 股）**——此類本就無「全體董監持股」資料，依**寧可漏報**原則排除為正確行為，故對「本國有董監申報之個股」的有效涵蓋率更高。上櫃缺漏 3 檔（6945／6983／7819，多為近期新上櫃）。

## 風險 / 注意

- **上櫃更新頻率（最主要殘留風險）**：上櫃董監明細（`t187ap11_O` / data.gov 22812）被標示為「每1年」，雖本次抓到的快照與上市同為 2026-04（新鮮）且為同一 MOPS 表單分市場版本（上市版確認每月），但「每1年」標籤未必是筆誤、無法在單次 POC 內證明每月刷新。**緩解：**（a）管線加「資料年月新鮮度守門」——若上櫃 `資料年月` 落後上市或距今 > 2 個月即告警；（b）保留 **MOPS 每月逐檔備援（架構 A，已驗證 `mopsov` 端點＋解析可行）**，必要時上櫃切換之。
- **發行股數特例**：TDR 原股、特別股、私募；本 POC 直接取已發行股數欄位（已避開資本額估算），但若未來條件需含特別股，需再界定。
- **姓名去重的罕見風險**：去重以 `姓名` 為鍵；同公司若有兩位同名自然人會被併為一筆（極罕見）。MOPS 以身分證／統編去重更嚴謹，可作為定期抽驗。
- **2317 鴻海 = 0.02%** 偏低但屬合理：席次多由不持大量股之自然人董事擔任，法人大股東未以法人董事身分入列（其代表人本就排除）。屬 in-range 正常輸出，非錯誤；可日後再抽驗一檔加強信心。
- **MOPS 反爬／主機遷移**：架構 A 為逐檔 POST、主機已遷移過一次（mops→mopsov）、舊主機回封鎖頁——大規模逐檔爬有風險，故主架構採批次明細（架構 B），MOPS 僅作 oracle/備援。
- **取 CSV 需送 `Accept: text/csv`**：OpenAPI 預設回 JSON；亦可改以 JSON 解析（欄位同）。

## 對主設計的影響（關閉 §11）

- **§11「董監持股確切來源端點與解析方式」已由本 POC 確定**：
  - 主來源＝TWSE/TPEx OpenAPI `t187ap11_L` / `mopsfin_t187ap11_O`（董監明細，`Accept: text/csv`）＋ `t187ap03_L` / `mopsfin_t187ap03_O`（已發行股數）。
  - 解析＝`parseDirectorRows`（只計董監本人、依姓名去重）＋ `aggregateByShares`，已逐股對齊 MOPS。
- **`director_holdings_monthly` 寫入流程**：管線「董監抓取」步驟改為**每月觸發 / 或每日檢查當月明細是否已公布**（出表日期約於次月中旬，如 2026-04 資料於 2026-05-20 出表）；寫入時記錄 `director_data_month`。加上述「新鮮度守門」。
- **上櫃**：先採架構 B（批次明細）＋新鮮度守門；若實際運行發現上櫃明細不按月刷新，切換 MOPS 架構 A（每月逐檔，已驗證可行）。
- 結論為 **go**，可回到 writing-plans 展開其餘完整實作計畫（資料管線、訊號計算、篩選引擎與 API、前端）。
